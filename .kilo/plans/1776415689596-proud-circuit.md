# Plan: Cookie-Based Auth Migration

## Goal
Migrasi dari token-based auth (localStorage) ke cookie-based auth dengan server-side validation.

---

## Files to Modify

### Backend (Go/Gin)

1. **`internal/handler/handlers.go`**
   - Update `Login()` - set HTTP-only cookie, hapus token dari response
   - Add `Logout()` - hapus cookie
   - Add `ValidateSession()` - return user info untuk auth check
   - Update `AuthMiddleware()` - baca token dari cookie, fallback ke header

2. **`main.go`**
   - Add route `POST /api/logout`
   - Add route `GET /api/auth/validate`

---

### Frontend (Svelte 5)

3. **`web/src/lib/stores.js`**
   - Hapus `user` dan `token` dari localStorage persistence
   - Tambah `isAuthenticated` state
   - Export `logout()` function

4. **`web/src/lib/api.js`**
   - Tambah `withCredentials: true`
   - Add `checkAuth()` function
   - Add `doLogout()` function

5. **`web/src/lib/auth.js`**
   - Update `protectRoute()` - async server validation menggunakan `checkAuth()`

6. **`web/src/routes/login/+page.svelte`**
   - Hapus `localStorage.setItem()` manual

7. **`web/src/routes/+layout.svelte`**
   - Gunakan `$isAuthenticated` untuk conditional rendering

---

## Implementation Order

1. Backend: Update `handlers.go` (Login, Logout, ValidateSession, AuthMiddleware)
2. Backend: Update `main.go` (add routes)
3. Frontend: Update `stores.js`
4. Frontend: Update `api.js`
5. Frontend: Update `auth.js`
6. Frontend: Update `login/+page.svelte`
7. Frontend: Update `+layout.svelte`
